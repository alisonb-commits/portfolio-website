Overview

A static blog that
actually does things

Most Hugo starters stop at rendering Markdown. This one goes further — pulling in live post data via a JavaScript API layer, letting visitors search and filter posts in the browser without a page reload, and opening full posts in a modal overlay rather than navigating away.

On the backend, Netlify handles form submissions and Zapier routes new contact entries to an automated email sequence — zero server required. The build pipeline chains npm scripts with Hugo's native build system, keeping the workflow clean and deployable with a single git push.

Client-Side Search

Instant filtering
no reload

A lightweight vanilla JS search layer reads post titles, tags, and summaries from a pre-built JSON index at build time. As the user types, results filter live — no external search service, no API round-trip.

This keeps the site fully static while still feeling interactive and responsive.

Post Modals

Click to read,
stay in place

Rather than navigating to a separate page, each post card triggers a modal overlay that renders the full post content. This reduces perceived load time and keeps the reading context anchored to the list view.

Built with vanilla JS — no libraries, no dependencies. Keyboard-accessible with focus trapping and escape-to-close.

Build Process

How it comes
together

The project is composed from four distinct layers — static site generation, styling, interactivity, and automation — all wired together through a single build command.

01
Hugo + npm pipeline
Hugo handles templating and content rendering. npm scripts orchestrate the build order — Tailwind compiles first, Hugo picks up the output CSS, and the final dist folder deploys directly to Netlify.
02
Tailwind CSS v4
v4's new CSS-first config removes the need for a JS config file. Utility classes compile to a minimal stylesheet with zero unused rules — keeping page weight low on a content-heavy site.
03
Netlify forms + Zapier
The contact form uses Netlify's built-in form detection — no backend code required. A Zapier zap listens to new Netlify form submissions and fires an automated email reply, creating a lightweight CRM-style follow-up flow.
04
Vanilla JS API integration
Post data is exposed as a JSON endpoint at build time. JavaScript fetches and renders this data client-side, enabling filtering and modal behaviour without React or any framework overhead.
