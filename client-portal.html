Structure First

Wireframe Exploration

Before building, the layout and content hierarchy were mapped through low-fidelity wireframes — defining the sidebar navigation model, the dashboard overview structure, and the CRUD workflow for clients and projects before any visual design decisions were made.

Security

Authentication & Access Control

The portal begins with a secure sign-in flow backed by JWT authentication, bcrypt password hashing, and HTTP security headers via Helmet. On login, a signed token is issued and persisted through React context, creating a clear boundary between public and protected surfaces.

Every backend API route validates the Bearer token before responding, and rate limiting on auth endpoints guards against brute-force attempts. Users can also register new accounts and update their profile through the same secured routes.

Core Product

Overview Dashboard

The dashboard gives a live snapshot of all active clients and work at a glance — total clients, total projects, and active project count surfaced immediately after login. The sidebar navigation provides fast access to all sections, and the recent clients list shows the latest activity without needing to navigate away.

The interface was designed with a clean, Notion-inspired aesthetic — warm off-white surfaces, minimal chrome, and clear typographic hierarchy — to feel familiar and focused rather than like a generic admin panel.

Client Management

Full CRUD with Search & Filter

The Clients page gives full control over every client record — create, view, edit, and delete, with real-time search filtering and status-based filtering available simultaneously. Each client links to their associated projects through a relational database structure.

The table view was kept deliberately lean: name with avatar initial, company, email, project count, and status badge. Everything a user needs to act, nothing that gets in the way.

Project Pipeline

Status Tracking & Inline Editing

Projects move through a three-stage pipeline — Pending, Active, and Completed — linked to their parent client through a relational PostgreSQL schema. The edit modal allows any field to be updated without leaving the page, with changes persisted immediately through protected API routes.

This interaction pattern mirrors what you'd find in real internal tools: fast, contextual edits within the current view rather than navigating to a separate detail page.

Account Settings

Profile & Responsive Design

The profile page lets users update their display name and change their password through separate, focused forms — each backed by a protected API route. Email is surfaced as read-only to reflect the auth constraint clearly rather than hiding it.

The entire application is fully responsive. On mobile, the sidebar navigation collapses behind a hamburger, stat cards stack vertically, and tables reflow — making the admin experience just as usable on a phone as on desktop.
